Week Four

The UI Era Begins
Date: 17.02.2025 - 22.02.2025

This week, I officially stepped into my UI & Interaction Systems Developer role.
It was a big shift from just planning to actively learning how to bring the interface to life.

Understanding My Role

Since UI is how players interact with the game, I spent time studying best practices for UI design, player experience, and Unity’s UI tools. I also looked into animations, event handling, and accessibility to make sure our game is intuitive and engaging.

Creating a Workable Timeline

With midterms approaching, our team decided to lighten the workload over the next two weeks. We made a development timeline to ensure we stay on track without burning out.

Research Over Implementation

So far, I haven’t built anything tangible, but I’ve been doing deep research into how UI elements should function. Next week, I’ll transition from research to building an actual prototype.

What I Learned / My Takeaways

  • Unity’s UI system is more complex than I thought, but very flexible.
  • UI isn’t just about visuals—it affects how players experience the game.
  • Planning around midterms is necessary to avoid getting overwhelmed.

Goals for Next Week

  1. Start implementing the basic inventory, task list, and dialogue UI.
  2. Experiment with button interactions, animations, and menu navigation.
  3. Make sure everything is structured well for future expansion.

Conclusion: A Productive Start

I’m feeling more motivated this week, even if I haven’t built anything yet. Now that I understand what I need to do, I can start bringing the UI to life next week.
